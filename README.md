# EXEC.SEQUENCE  
**Domain: protocols.exec**  
**Version: 1.0**

`EXEC.SEQUENCE` defines how the AIC‑HMV architecture executes ordered chains of operators with deterministic state transitions and full observability.

---

## 1. Purpose

`EXEC.SEQUENCE` enables:

- Multi‑step execution flows  
- Deterministic ordering  
- State‑safe transitions  
- Automatic trace propagation  
- Recovery on failure  

It is the backbone of complex operational behavior.

---

## 2. Invocation

EXEC::SEQUENCE.run steps=<list> [mode=strict|relaxed]

Example:

EXEC::SEQUENCE.run steps=[SCAN::GLOBAL, STATE::WRITE.delta +integrity]

---

## 3. Preconditions

- System must be initialized (`INIT.SYSTEM`)  
- Observability must be active  
- Steps must resolve to valid operators  

---

## 4. Sequence Model

A sequence is defined as:

SEQUENCE {
id: <uuid>
steps: [<operator>...]
mode: strict|relaxed
}

### Modes:
- **strict** — all steps must succeed  
- **relaxed** — failures allowed, but traced  

---

## 5. Execution Flow

1. Validate step list  
2. Emit `TRACE.SPAN` for sequence start  
3. Execute each step in order  
4. Emit state deltas  
5. Emit completion span  

---

## 6. Failure Modes

- `ERR.SEQUENCE.INVALID_STEP`  
- `ERR.SEQUENCE.HALT`  
- `ERR.SEQUENCE.UNRESOLVED_OPERATOR`  

---

**End of EXEC.SEQUENCE**
